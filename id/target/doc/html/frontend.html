iDMA Frontend

The frontend is responsible for providing a configuration interface for the DMA to the rest of the system. The frontend subsequently translates the system-specific configuration commands to the DMA’s 1D or ND request interfaces.

The frontend provides a configuration interface for the iDMA for various platforms. Currently the following three frontends are planned and in development:

  • Register Frontends: Register-based configuration interface

  • Snitch: Snitch integration

  • Ariane/Linux: An Ariane interface to allow use in a Linux system (not a priority yet)
